GIT

Git is a distributed version control system used to track changes in source code, manage different versions of a project, and collaborate with others. It allows developers to work simultaneously on a project, create branches for new features, merge changes, and maintain a complete history of all modifications efficiently.
  • Features of GIT
    1. Distributed Version Control
      2. Every developer has a complete copy of the repository, allowing work offline without relying on a central server.
    2. Branching and Merging
      3. Easily create branches to work on features or fixes and merge them without losing data.
    3. Collaboration
      4. Enables multiple developers to work on the same project seamlessly using platforms like GitHub and GitLab.
  • Basic Git Workflow
    1. Create / Clone
      2. Make a new repo or copy an existing one.
    2. Modify Files
      3. Edit, add, or delete code.
    3. Stage Changes
      4. Select files to commit using git add.
    4. Commit
      5. Save a snapshot using git commit.
    5. Push / Pull
      6. Upload changes to remote or download updates.
